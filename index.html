python：不可变数据

不可变数据类型的内容在创建后无法修改，这对于编写安全、可预测的代码非常有用。Python支持一些不可变的数据类型和相关技术来实现不可变数据（Immutable Data）。

不可变数据类型

Python 内置了几种不可变数据类型：元组（Tuple），字符串（String），frozenset，数字（Number）。

元组（Tuple）

元组是不可变的序列。一旦创建，元组的内容无法更改。

命名元组(namedtuple)

Python 中的namedtuple也是不可变的。一旦创建了namedtuple的实例，其字段就无法修改。namedtuple不变性的主要特征：

  1. 不能给字段赋值：这样做会引发 AttributeError
  2. 字段固定：namedtuple实例创建后，字段值就固定了，如果需要修改，必须创建新的实例。
  3. namedtuple不变性使命名元组可哈希化（如果所有字段都是可哈希化的），因可以将其用作字典中的键或将其存储在集合中。它确保数据一致性，因为值不会被意外修改。

数据类(dataclass)

在 Python 中，数据类是否不可变取决于其定义方式。

  1. 默认情况下，数据类实例是可变的，但
  2. 可以通过在 @dataclass 装饰器中设置 frosty=True 参数使其不可变。

事实上，当frosty=True的时候，数据类通过覆盖 setattr 方法来防止属性赋值。如果尝试修改属性，则会引发FrozenInstanceError。在这种设计下数据类和命名元组功能相似。

对不可变数据类而言，如果所有字段均可哈希，则该类型的实例自动可哈希。如果是可变数据类，除非明确设置unsafe_hash=True，否则不可哈希。

字符串（String）

Python的字符串是不可变的。任何修改字符串的操作都会生成一个新的字符串对象。

总结

不可变数据是函数式编程核心原则之一，它能够提高代码的可靠性、一致性和可预测性。不可变数据在python函数式编程中可以：

  1. 消除副作用：在函数式编程中要求纯函数的输出只依赖于输入，没有副作用。不可变数据确保函数不能意外修改外部状态，数据在程序的执行过程中保持一致和可预测。
  2. 线程安全：不可变数据结构可以在多个线程间自由共享，而不会产生数据竞争或不一致。这消除了使用同步机制（如锁）的需求，使并发编程更加简单和高效。
  3. 可预测性：不可变数据确保数据不会意外更改，这使得代码更容易推理。可预测性带来的好处包括不需要追踪数据在程序中何时、何地被修改，并可以轻松复现和验证程序的状态。
  4. 促进引用透明性：函数调用可以被它的结果替换，而不会改变程序行为。对同一输入的多次函数调用总是产生相同的结果，同时缓存和记忆化（memoization）更可靠，因为数据不会发生变化。
  5. 简化调试和测试：使用不可变数据使得程序状态的快照（如用于调试）在程序运行过程中始终有效，测试用例更简单，因为不需要考虑意外的数据修改
  6. 方便实现撤销和重做：应用程序的每个“状态”都可以被保存为一个新的副本，可以轻松在不同状态间切换，而不丢失数据。
  7. 符合函数式编程的范式：不可变数据强化了函数式编程的声明式特点，更关注“做什么”而不是“如何做”，函数对数据进行转换，而不是直接修改。

python: 高阶函数

在python语法里，函数是一等公民，支持高阶函数。高阶函数（Higher-Order Function）是指以下两类函数：

  1. 接受一个或多个函数作为参数的函数。
  2. 将一个函数作为结果返回的函数。

函数对象作为函数参数

我们可以将一个函数作为参数传递给另一个函数。例如，将一个函数应用到列表中的每个元素。这时候，lambda经常被用来创建匿名函数传递给另一函数。

Python内置高阶函数

Python 提供了一些常见的高阶函数，例如 map、filter 和 reduce。

  1. map ：将一个函数应用于可迭代对象的每个元素，返回一个包含结果的新迭代器。
  2. filter ：使用一个函数来过滤可迭代对象中的元素，保留返回值为 True 的元素。
  3. reduce ：使用一个二元函数将可迭代对象缩减为单一值。

总结

高阶函数有许多优点，是 Python 函数式编程的核心，能够帮助开发者编写优雅、简洁且具有表现力的代码

  1. 代码复用性：将逻辑抽象为可重用的函数。
  2. 代码简洁性：关注“做什么”而不是“怎么做”。
  3. 不变性：鼓励函数式编程的原则，比如不变性和无状态性

python: 编写纯函数

什么是纯函数？

纯函数是指只依赖输入参数并返回一个结果的函数，而不产生任何副作用（如修改全局变量或输出到控制台）。

  1. 依赖输入：输出完全由输入参数决定。
  2. 不依赖外部状态：不访问或修改全局变量、类属性或函数范围之外的任何可变数据。
  3. 无副作用：不会打印到控制台、写入文件、发出网络请求或修改输入参数。
  4. 引用透明性：函数调用可以用其返回值替换，而不会改变程序的行为。

为什么代码要写成纯函数？

纯函数的优点是相同的输入总是返回相同的输出，使得代码更容易理解、调试和测试。

  1. 可测试性：纯函数更易于测试，因为它们的输出可以根据输入进行预测。
  2. 可调试性：使用纯函数的代码更容易推理和调试，因为它们的行为是独立的，不依赖于外部因素。
  3. 并行性：纯函数可以轻松并行化，因为它们没有可能导致竞争条件的副作用。
  4. 可缓存性：纯函数调用的结果可以缓存（记忆）以提高性能。
  5. 可读性和可维护性：使用纯函数的代码通常更容易理解和维护，因为它更模块化且更可预测。

如何编写纯函数？

  1. 避免修改输入参数：如果您需要修改数据结构，请在函数内创建它的副本并修改副本。
  2. 不使用全局变量：如果您需要访问外部数据，请将其作为参数传递给函数。
  3. 避免副作用：不要执行任何修改外部状态的操作，例如打印到控制台或写入文件。
  4. 确保确定性行为：确保函数始终为相同的输入返回相同的输出。

python: 函数式编程

函数式编程是一种编程模式，强调下面几个特点：

  1. 编写无副作用的函数（纯函数）：给定相同的输入总是返回相同的输出，并且不产生任何副作用
  2. 无状态函数：应尽量避免依赖任何外部状态或上下文，而是完全依赖输入参数
  3. 函数作为一等公民，使用高阶函数：接受函数作为参数，或返回函数，比如 map、filter、reduce。
  4. 使用不可变数据结构
  5. 递归：递归与不可变数据结构结合，使得代码更具声明性，便于理解
  6. Monads：使用Monads处理副作用（如异常、状态变化、I/O 等）

虽然python不是为函数式编程设计的，但是python支持很多函数式编程的语法。下面一系列的文章中，我结合一些例子讨论函数式编程在Python开发中的应用。

  1. 在python语法里，函数是一等公民，支持高阶函数，python可以将函数作为参数传递，或者将函数作为返回值。 python支持Lambda

  2. python支持递归

  3. python支持函数式工具（map、filter、reduce），对可迭代对象进行映射、过滤和聚合操作

  4. python支持列表推导式与生成器表达式

  5. python支持不可变数据结构，比如tuple和frozenset

  6. python支持函数式工具模块 (functools)和itertools 模块

  7. python支持闭包与装饰器，闭包允许函数内部定义函数并捕获外部变量；装饰器可以用于对函数进行功能增强。

  8. python支持Immutable Data（如 dataclasses 与 types.MappingProxyType）
